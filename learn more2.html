2016:This year decided to work on implementation of the project one step ahead. Main focus was in four areas: Website, server, database and scraper.

So we populated the main project on github https://github.com/AHOCHI for easy use to all team members. We broke the whole part into four repositories

1. AHOCHI-server 2. AHOCHI-crawler 3.AHOCHI-Site 4. AHOCHI.github.io

1. AHOCHI-server: This is server part of AHOCHI.Basic purpose of this server is to maintain the API for service providers. Team decided to work on MongoDB.( MongoDB is an open source database that uses a document-oriented data model. MongoDB is one of several database types to arise in the mid-2000s under the NoSQL banner. Instead of using tables and rows as in relational databases, MongoDB is built on an architecture of collections and documents).Basic CRUD operation was used for server API. It directly handle records or data objects; apart from these operations, the records are passive entities for this project. Typically it's just database tables and records to save relevant content or data for the project AHOCHI. This is capable to the queries the data by state, zip, city, and county, as well as text search.

2. AHOCHI-crawler: The basic purpose of the web crawler is to finds services and their location and stores that information in the database in this project. Currently being developed using PyCharm Community Edition 2016 . Crawler will run across the various websites to search for potential charities and upload them to the AHOCHI server. Web Crawler or Spider is a perfect platform for crawling the data. A Search Engine Spider (also known as a crawler, Robot, SearchBot or simply a Bot) is a program that most search engines use to find what’s new on the Internet. There are many types of web spiders in use, but for now, we’re only interested in the Bot that actually “crawls” the web and collects documents to build a searchable index for the different search engines. The program starts at a website and follows every hyperlink on each page for potential services and charities in this part. So there are basically three steps that are involved in the web crawling procedure for AHOCHI. First, the search bot starts by crawling the pages of the site. Then it continues indexing the words and content of the site such as senior citizen care, and finally it visit the links (web page addresses or URLs) that are found in our site. When the spider doesn’t find a page, it will eventually be deleted from the index. However, some of the spiders will check again for a second time to verify that the page really is offline. If it finds the page for relevant information it will save to database.
